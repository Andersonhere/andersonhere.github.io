# 博客使用说明（维护汇总）

本文档汇总本仓库 Jekyll + Theme Yat 的约定与常见操作，便于后续自行补充。对应实现已落在 `_config.yml`、`.github/workflows/`、`_includes/`、`_scripts/` 等处。

---

## 1. 部署（GitHub Pages + Actions）

- **仓库**：用户站根仓库（如 `username.github.io`）；若在单仓子目录开发，需把变更推送到该根仓库，Pages 只认该仓库。
- **Settings → Pages**：**Source** 选 **GitHub Actions**（不要再用「从分支部署」）。
- **工作流**：`.github/workflows/jekyll-gh-pages.yml`  
  - 使用 `actions/configure-pages@v6`、`upload-pages-artifact@v5`、`deploy-pages@v5`（主版本号由官方维护，避免引用不存在的 patch tag）。
- **`vendor`**：已在 `_config.yml` 的 `exclude` 中排除；CI 上 `bundler-cache` 会把 gem 装进 `vendor/bundle`，不排除时 Jekyll 会误扫 gem 内模板并报错。

---

## 2. Ruby / Gem 与构建环境

- **Gemfile**：使用 **Jekyll 4** + `jekyll-sass-converter` 3.x（Dart Sass），以兼容 Yat 主题里 `@use` / `sass:meta` 等样式；不再使用整包 `github-pages`（其锁在 Jekyll 3.9，会编译 SCSS 失败）。
- **插件**：`_config.yml` 中 `plugins` 与 `Gemfile` 的 `:jekyll_plugins` 保持一致；含 **`jekyll-spaceship`** 时需在两边都声明。
- **本地**：可用 `bundle install` + `bundle exec jekyll serve`；与 CI 对齐时建议 Ruby **3.3**（与 workflow 一致）。

---

## 3. `_config.yml` 关键约定

| 项 | 说明 |
|----|------|
| **`defaults` → `posts`** | `layout: post`：文章未写 `layout` 时也会走主题正文样式。 |
| **`permalink`** | 必须写在 **`defaults` 里且仅 `type: posts`**，**不要**写在顶层。顶层 `permalink` 会改掉主题的 `categories.html` / `tags.html` / `archives.html` 等地址，侧栏仍指向 `/categories.html` → **整页 404**。 |
| **`paginate` / `paginate_path`** | 首页列表分页；`paginate_path` 默认 `/page:num/`（第 2 页起 `/page2/` …）。 |
| **`exclude`** | 除 `vendor`、锁文件等外，可将本说明、`jekyll-theme-yat-setup.md` 等**不参与站点**的 Markdown 列入，避免被当成页面生成。 |

---

## 4. 文章（`_posts`）Front Matter 常用项

- **`title`**：标题；与 `permalink: /:title/` 一起决定文章 URL（slug 规则见 Jekyll 文档）。
- **`date`**：发布时间；**同一天多篇**时，仅靠文件名字符串排序会乱序；多章节系列可用 **不同时间**（例如 `2026-04-17 00:09:00 +0800` 表示第 9 章）保证 `上一篇/下一篇` 正确。
- **`categories` / `tags`**：分类与标签；首页侧栏分类依赖 `categories`。
- **`excerpt_image`**：列表缩略图（推荐）。也可用 **`banner`** 或 `banner.image`；不配时用主题默认图。若**不要列表图**：`excerpt_image: NO_EXCERPT_IMAGE`。
- 全站默认列表图：在 `_config.yml` 的 `yat.excerpt.default_image` 中配置（见主题说明）。

**不要**在正文里用 `./08-xxx.md` 这类**源文件相对链接**当「上一章/下一章」：站点 URL 是 `/:title/`，相对 `.md` 会 404；章节导航应交给主题底部 **`page.previous` / `page.next`**（依赖 `date` 排序）。

---

## 5. 主题覆盖文件（本仓库 `_includes`）

按需维护，升级主题时注意 diff：

| 路径 | 作用 |
|------|------|
| `_includes/views/segments.html` | 分段标题 `id` 与侧栏 `common-list` 的 `#h-...` 锚点一致（标签/分类/归档页内跳转）。 |
| `_includes/views/paginator.html` | 首页分页：**上一页/下一页**、**页码**、**跳转**；保留 `previous` / `next` / `indicator` 类名以兼容主题 CSS。 |
| `_includes/custom-head.html` | 全站额外 `<style>`（如代码块、表格微调）。 |

---

## 6. 脚本

- **`_scripts/fix_post_nav_dates.py`**：批量处理「第 N 章」系列文件名，写入 `date: ... 00:NN:00`（NN=章号），并删除手写「上一章/下一章」行（支持中文冒号）。可按需修改后复跑。

---

## 7. 其他说明

- **`url` / `repository`**：与真实 GitHub Pages 域名、仓库名一致，便于 feed、SEO、主题内链。
- **欢迎文等非章节文**：若需固定出现在章节链**末尾**，可把 `date` 调到全系列章节之后（例如同日更晚一小时），避免插在章节「下一篇」中间。
- **Node / Actions 弃用提示**：跟随官方将 Pages 相关 action 升到带 Node 24 的主版本即可；不必长期手写不存在的 `v4.0.6` 类 tag。
