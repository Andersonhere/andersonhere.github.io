> **核心问题**：磁盘是如何工作的？

## 本章大纲

### 37.1 接口

**磁盘接口**：
- 扇区：最小读写单位（通常 512 字节或 4KB）
- 地址：柱面、磁头、扇区 (CHS) 或逻辑块地址 (LBA)

### 37.2 基本几何形状

```
磁盘结构:

        ┌──────────────────────┐
        │      磁盘片          │
        │  ┌────────────────┐  │
        │  │    磁道        │  │
        │  │  ┌──────────┐  │  │
        │  │  │  扇区    │  │  │
        │  │  │  ┌────┐  │  │  │
        │  │  │  │扇区│  │  │  │
        │  │  │  └────┘  │  │  │
        │  │  └──────────┘  │  │
        │  └────────────────┘  │
        └──────────────────────┘
              ↑
           磁头
```

| 组件 | 说明 |
|------|------|
| 盘片 | 存储数据的圆形磁盘 |
| 磁道 | 盘片上的同心圆 |
| 扇区 | 磁道上的数据单元 |
| 磁头 | 读写数据的设备 |

### 37.3 简单的磁盘驱动器

**访问时间组成**：

```
总访问时间 = 寻道时间 + 旋转延迟 + 传输时间

寻道时间: 磁头移动到目标磁道
旋转延迟: 等待目标扇区转到磁头下
传输时间: 读写数据的时间
```

### 37.4 I/O 时间：用数学

**计算示例**：

```
假设:
- 平均寻道时间: 5ms
- 转速: 7200 RPM (旋转一周 = 8.33ms)
- 平均旋转延迟: 4.17ms
- 传输速率: 100MB/s

读取 4KB 数据:
- 寻道: 5ms
- 旋转延迟: 4.17ms
- 传输: 4KB / 100MB/s = 0.04ms

总计: ≈ 9.2ms
```

### 37.5 磁盘调度

#### 先进先出 (FIFO)

```
请求顺序: 读取
优点: 公平
缺点: 未优化性能
```

#### 最短寻道时间优先 (SSTF)

```
选择离当前磁头位置最近的请求
优点: 减少寻道时间
缺点: 可能饥饿远端请求
```

#### 电梯算法 (SCAN/C-SCAN)

```
SCAN: 磁头单向移动，处理沿途请求
C-SCAN: 循环扫描，只在一个方向处理请求

优点: 公平，无饥饿
缺点: 平均等待时间可能较长
```

### 37.6 小结

**磁盘性能因素**：

| 因素 | 影响 |
|------|------|
| 转速 | 影响旋转延迟 |
| 寻道时间 | 影响访问时间 |
| 缓存 | 减少实际磁盘访问 |
| 调度算法 | 影响平均等待时间 |

## 磁盘调度对比

```
请求队列: 23, 89, 132, 42, 187
当前磁头: 100

FIFO:  100 → 23 → 89 → 132 → 42 → 187
SSTF:  100 → 89 → 132 → 42 → 23 → 187
SCAN:  100 → 132 → 187 → 89 → 42 → 23
```
