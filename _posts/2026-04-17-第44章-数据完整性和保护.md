> **核心问题**：如何保护数据不被损坏？

## 本章大纲

### 44.1 磁盘故障模式

| 模式 | 说明 |
|------|------|
| 整体故障 | 磁盘完全停止工作 |
| 部分故障 | 部分扇区损坏 |
| 静默损坏 | 数据错误但磁盘继续工作 |

### 44.2 处理潜在的扇区错误

**检测**：磁盘报告读取错误

**处理**：
- 使用冗余数据恢复
- 标记扇区为坏块

### 44.3 检测讹误：校验和

**校验和**：数据的简短指纹

```c
// 简单校验和
uint8_t checksum(char *data, int len) {
    uint8_t sum = 0;
    for (int i = 0; i < len; i++) {
        sum += data[i];
    }
    return sum;
}
```

**更好的选择**：

| 算法 | 特点 |
|------|------|
| 简单求和 | 快，但碰撞率高 |
| CRC | 检测突发错误 |
| MD5/SHA | 加密强度，检测恶意篡改 |

### 44.4 使用校验和

**存储方式**：

```
每个块附带校验和:
┌────────────────────────────┐
│ 数据块 (4KB) │ 校验和 (8B) │
└────────────────────────────┘

读取时验证:
data = read_block()
if (checksum(data) != stored_checksum) {
    error("数据损坏");
}
```

### 44.5 一个新问题：错误的写入

**问题**：写入到错误位置

```
预期: 写入块 100
实际: 写入块 101

校验和验证通过，但数据在错误位置！
```

**解决方案**：存储物理标识

```
┌────────────────────────────────┐
│ 数据块 │ 校验和 │ 物理块号    │
└────────────────────────────────┘
```

### 44.6 最后一个问题：丢失的写入

**问题**：写入从未到达磁盘

```
报告写入成功，但实际未写入

检测方式:
- 写入后立即读取验证
- 使用额外标记
```

### 44.7 擦净

**定期扫描**：检测所有块的完整性

```
1. 读取所有块
2. 验证校验和
3. 尝试恢复损坏数据
4. 报告不可恢复错误
```

### 44.8 校验和的开销

| 开销 | 说明 |
|------|------|
| CPU | 计算校验和 |
| 存储 | 存储校验和 |
| I/O | 读取时验证 |

### 44.9 小结

**数据保护策略**：

| 层次 | 机制 |
|------|------|
| 磁盘层 | 内部错误检测 |
| 系统层 | 校验和 |
| 应用层 | 数据验证 |

**最佳实践**：

1. 使用校验和检测损坏
2. 使用冗余恢复数据
3. 定期擦净检测潜在问题
4. 备份重要数据

## 思考题

1. 为什么校验和不能防止所有错误？
2. 如何选择合适的校验和算法？
