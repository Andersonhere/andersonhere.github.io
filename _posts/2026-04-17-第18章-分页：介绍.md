> **核心问题**：如何灵活地管理虚拟内存？

## 本章大纲

### 18.1 一个简单例子

**分页核心思想**：
- 将地址空间分成固定大小的页
- 将物理内存分成相同大小的页框
- 每个页可以映射到任意页框

```
虚拟地址空间 (64字节, 16字节/页)
┌─────────┐
│ 页0     │ → 页框3
├─────────┤
│ 页1     │ → 页框7
├─────────┤
│ 页2     │ → 页框5
├─────────┤
│ 页3     │ → 页框2
└─────────┘

虚拟地址: [页号 | 偏移]
         [ 2位  | 4位 ]  (64字节地址空间, 16字节页)
```

### 18.2 页表存在哪里

**存储位置**：

| 位置 | 优点 | 缺点 |
|------|------|------|
| 内存 | 灵活 | 访问慢(需额外内存访问) |
| 专用硬件 | 快 | 容量有限 |
| TLB | 极快 | 容量小 |

### 18.3 列表中究竟有什么

**页表项 (PTE) 结构**：

```
┌─────────────────────────────────────────────┐
│ 有效位 │ 保护位 │ 脏位 │ 访问位 │ 物理页号 │
└─────────────────────────────────────────────┘

有效位: 该页是否在内存中
保护位: 读/写/执行权限
脏位:  页是否被修改
访问位: 页是否被访问(用于替换)
物理页号: 实际物理页框号
```

### 18.4 分页：也很慢

**问题**：每次内存访问需要两次内存访问

```
访问虚拟地址 100:
  1. 访问页表获取物理页号
  2. 访问实际物理地址
```

**解决方案**：TLB (Translation Lookaside Buffer)

### 18.5 内存追踪

**完整地址转换流程**：

```
虚拟地址
    ↓
[VPN | Offset]  提取虚拟页号
    ↓
  TLB 查找
    ↓
  ┌───┴───┐
  命中    未命中
    ↓       ↓
  PFN    访问页表
          ↓
        更新TLB
          ↓
  ←───────┘
    ↓
物理地址 = PFN * PageSize + Offset
```

### 18.6 小结

**分页优缺点**：

| 优点 | 缺点 |
|------|------|
| 无外部碎片 | 页表占用大 |
| 支持稀疏地址空间 | 地址转换开销 |
| 易于共享和保护 | 需要硬件支持 |
