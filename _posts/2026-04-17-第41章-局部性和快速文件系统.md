> **核心问题**：如何优化文件系统性能？

## 本章大纲

### 41.1 问题：性能不佳

**早期 Unix 文件系统 (FFS) 的问题**：

| 问题 | 原因 |
|------|------|
| 碎片化 | 文件分散在磁盘各处 |
| 小块分配 | 导致大量随机 I/O |
| 性能差 | 寻道和旋转延迟 |

### 41.2 FFS：磁盘意识是解决方案

**核心思想**：让文件系统理解磁盘几何结构

### 41.3 组织结构：柱面组

```
磁盘组织为柱面组:

柱面组0          柱面组1          柱面组2
┌────────┐      ┌────────┐      ┌────────┐
│超级块   │      │超级块   │      │超级块   │
│inode区 │      │inode区 │      │inode区 │
│数据块   │      │数据块   │      │数据块   │
└────────┘      └────────┘      └────────┘
```

### 41.4 策略：如何分配文件和目录

**基本策略**：

1. **相关数据放在一起**
   - 目录和其文件放在同一柱面组
   - 文件的数据块尽量连续

2. **大文件分散放置**
   - 大文件跨多个柱面组
   - 避免填满单一柱面组

**分配算法**：

```
创建文件:
1. 在与目录相同的柱面组分配 inode
2. 尝试在相同柱面组分配数据块
3. 如果柱面组空间不足，选择附近柱面组

大文件:
1. 前几个块放在起始柱面组
2. 后续块放在其他柱面组（每几 MB 切换）
```

### 41.5 测量文件的局部性

**顺序访问优化**：

```
理想布局:
块1 → 块2 → 块3 → 块4
↓     ↓     ↓     ↓
连续扇区，无需寻道

糟糕布局:
块1 → ... → 块2 → ... → 块3 → ... → 块4
      寻道       寻道       寻道
```

### 41.6 大文件例外

**参数选择**：

```
大文件参数: 每 X MB 切换柱面组

目的:
- 避免大文件独占柱面组
- 保持小块文件的局部性
```

### 41.7 关于 FFS 的其他几件事

#### 预分配

```c
// 预分配连续块
posix_fallocate(fd, offset, len);
```

#### 子块分配

```
小块文件优化:
- 将块分成更小的片段 (如 512B)
- 减少小文件的空间浪费
```

#### 参数化

```
根据磁盘特性调整:
- 柱面组大小
- 每组 inode 数量
- 预分配策略
```

### 41.8 小结

**FFS 优化总结**：

| 优化 | 效果 |
|------|------|
| 柱面组 | 减少寻道 |
| 相关数据集中 | 提高局部性 |
| 大文件分散 | 避免独占 |
| 预分配 | 减少碎片 |
