> **核心问题**：如何选择换出哪个页？

## 本章大纲

### 22.1 缓存管理

**内存作为页的缓存**

```
                    ┌──────────┐
                    │  磁盘    │ ← 主存储
                    └────┬─────┘
                         │ 换入/换出
                    ┌────┴─────┐
                    │ 物理内存 │ ← 缓存
                    └──────────┘
```

**目标**：最小化缓存未命中

### 22.2 最优替换策略

**策略**：替换将来最久不会被访问的页

**特点**：
- 理论最优
- 无法实现（需要预知未来）
- 作为评价基准

### 22.3 简单策略：FIFO

**策略**：替换最早进入的页

```
页面访问序列: 0, 1, 2, 0, 1, 3, 0, 3, 1, 2, 1
缓存大小: 3

时间:  0   1   2   0   1   3   0   3   1   2   1
缓存: [0] [0,1] [0,1,2] [0,1,2] [0,1,2] [1,2,3] [2,3,0] [2,3,0] [3,0,1] [0,1,2] [0,1,2]
命中:      -     -      H      H     M      M      H      H     M      M      H

命中率: 5/11 = 45%
```

**优点**：简单
**缺点**：可能替换常用页

### 22.4 另一简单策略：随机

**策略**：随机选择替换

**特点**：
- 实现简单
- 性能不稳定
- 可复现性差

### 22.5 利用历史数据：LRU

**策略**：替换最近最少使用的页

**原理**：局部性原理

```
页面访问序列: 0, 1, 2, 0, 1, 3, 0, 3, 1, 2, 1
缓存大小: 3

LRU 假设最近访问的页将来也会被访问
```

**实现方式**：
- 时间戳：每次访问更新时间
- 链表：访问时移到头部，替换尾部

### 22.6 工作负载示例

| 负载类型 | 最佳策略 |
|----------|----------|
| 顺序访问 | FIFO/随机 |
| 循环访问 | 随机 |
| 局部性强 | LRU |

### 22.7 实现基于历史信息的算法

**精确 LRU 开销大**：

```
每次访问需要:
- 更新时间戳或移动链表
- 需要 O(1) 或 O(log n) 操作
```

### 22.8 近似 LRU

**时钟算法 (Clock Algorithm)**：

```
访问位使用情况:
     ┌───┐
     │ 0 │ ──→ ┌───┐
     └───┘     │ 1 │
       ↑       └───┘
       │         │
     ┌───┐     ┌───┐
     │ 1 │ ←── │ 0 │
     └───┘     └───┘

指针扫描，遇到访问位=0则替换
遇到访问位=1则清零继续
```

### 22.9 考虑脏页

**优化**：优先替换干净页（未修改）

```
替换干净页: 只需覆盖
替换脏页: 需要先写回磁盘
```

### 22.11 抖动

**问题**：内存不足以容纳工作集

**现象**：频繁换入换出，系统停滞

**解决方案**：
- 减少运行进程数
- 增加物理内存
- 调整工作集

### 22.12 小结

| 策略 | 实现 | 性能 |
|------|------|------|
| 最优 | 不可行 | 最优 |
| FIFO | 简单 | 较差 |
| 随机 | 简单 | 一般 |
| LRU | 复杂 | 好 |
| 近似LRU | 中等 | 较好 |
