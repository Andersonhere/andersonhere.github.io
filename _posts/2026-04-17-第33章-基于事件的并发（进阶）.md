> **核心问题**：不使用线程如何处理并发？

## 本章大纲

### 33.1 基本想法：事件循环

```c
while (1) {
    events = get_events();
    for (e in events) {
        process_event(e);
    }
}
```

**特点**：
- 单线程执行
- 无并发问题
- 非阻塞 I/O

### 33.2 重要 API：select()（或 poll()）

```c
int select(int nfds,
           fd_set *readfds,
           fd_set *writefds,
           fd_set *exceptfds,
           struct timeval *timeout);
```

**使用示例**：

```c
fd_set readfds;
FD_ZERO(&readfds);
FD_SET(socket_fd, &readfds);

int n = select(socket_fd + 1, &readfds, NULL, NULL, NULL);
if (n > 0 && FD_ISSET(socket_fd, &readfds)) {
    // socket 可读
    handle_read(socket_fd);
}
```

### 33.3 使用 select()

```c
void main_loop() {
    while (1) {
        fd_set readfds;
        FD_ZERO(&readfds);
        
        // 添加所有监听的 fd
        for (int i = 0; i < n_fds; i++) {
            FD_SET(fds[i], &readfds);
        }
        
        select(max_fd + 1, &readfds, NULL, NULL, NULL);
        
        // 处理就绪的 fd
        for (int i = 0; i < n_fds; i++) {
            if (FD_ISSET(fds[i], &readfds)) {
                handle_event(fds[i]);
            }
        }
    }
}
```

### 33.4 为何更简单？无须锁

**优势**：
- 单线程执行
- 无共享数据竞争
- 无死锁可能
- 易于调试

### 33.5 一个问题：阻塞系统调用

**问题**：阻塞调用会阻塞整个事件循环

```c
// 错误：会阻塞
void handle_client(int fd) {
    char buf[1024];
    read(fd, buf, 1024);  // 阻塞！
    process(buf);
}
```

### 33.6 解决方案：异步 I/O

```c
// 异步读取
aio_read(&aiocb);  // 立即返回

// 检查完成
while (aio_error(&aiocb) == EINPROGRESS)
    ; // 可以做其他事

// 完成
int bytes = aio_return(&aiocb);
```

### 33.7 另一个问题：状态管理

**问题**：无法使用栈保存状态

```c
// 多线程：状态在栈上
void handle() {
    State s;  // 栈上状态
    read(fd, &s.buf, ...);
    process(&s);
}  // 自动清理

// 事件驱动：状态需要手动管理
void handle_step1() {
    State *s = malloc(sizeof(State));
    aio_read(&s->aiocb);
    s->callback = handle_step2;  // 下一步回调
}
```

### 33.8 什么事情仍然很难

| 困难 | 说明 |
|------|------|
| 状态管理 | 需要手动管理状态机 |
| 代码复杂 | 回调地狱 |
| CPU 密集型任务 | 会阻塞事件循环 |
| 多核利用 | 需要多进程 |

### 33.9 小结

**事件驱动 vs 多线程**：

| 方面 | 事件驱动 | 多线程 |
|------|----------|--------|
| 复杂性 | 状态管理复杂 | 同步复杂 |
| 性能 | 单核效率高 | 多核利用率高 |
| 调试 | 较简单 | 较困难 |
| 适用场景 | I/O 密集 | CPU + I/O |
