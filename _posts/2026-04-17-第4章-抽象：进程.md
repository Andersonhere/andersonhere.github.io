> **核心问题**：如何提供"有许多CPU"的假象？

## 本章大纲

### 4.1 抽象：进程
- **进程定义**：运行中的程序
- 操作系统提供的核心抽象

### 4.2 进程 API
| API | 功能 |
|-----|------|
| `fork()` | 创建新进程 |
| `wait()` | 等待子进程结束 |
| `exec()` | 执行新程序 |

### 4.3 进程创建：更多细节

1. **加载代码和数据**到内存
2. **分配栈空间**（用于局部变量、函数调用）
3. **分配堆空间**（动态分配内存）
4. **初始化 I/O**（文件描述符 0, 1, 2）
5. **跳转到 main()** 开始执行

```
进程地址空间布局：
┌──────────────┐ 高地址
│    栈 ↓      │ 局部变量、函数调用
├──────────────┤
│     ...      │
├──────────────┤
│     堆 ↑     │ 动态分配内存
├──────────────┤
│   数据段     │ 全局变量、静态变量
├──────────────┤
│   代码段     │ 程序指令
└──────────────┘ 低地址
```

### 4.4 进程状态

```
         ┌────────────┐
         │   创建     │
         └─────┬──────┘
               ↓
         ┌────────────┐
    ┌───→│   就绪     │←───┐
    │    └─────┬──────┘    │
    │          ↓           │
    │    ┌────────────┐    │
    │    │   运行     │────┘
    │    └─────┬──────┘
    │          │ I/O等待
    │          ↓
    │    ┌────────────┐
    └────│   阻塞     │
         └────────────┘
```

| 状态 | 说明 |
|------|------|
| 运行 (Running) | 正在 CPU 上执行 |
| 就绪 (Ready) | 准备好运行，等待调度 |
| 阻塞 (Blocked) | 等待 I/O 或其他事件 |

### 4.5 数据结构

进程控制块 (PCB) / 任务结构 (task_struct) 包含：

- 进程 ID (PID)
- 寄存器状态
- 内存管理信息
- 打开的文件列表
- 进程状态
- 调度信息

### 4.6 小结

进程是操作系统的核心抽象：
- 提供运行程序的独立环境
- 通过时分共享实现多进程并发
- 需要硬件和软件协同支持

## 关键概念

| 概念 | 定义 |
|------|------|
| 进程 | 运行中的程序 |
| 地址空间 | 进程可访问的内存范围 |
| 上下文切换 | 切换正在运行的进程 |
| PCB | 存储进程信息的数据结构 |
