> **核心问题**：如何实现正确的锁？

## 本章大纲

### 28.1 锁的基本思想

**锁的目的**：提供互斥访问

```c
lock(&m);
// 临界区：只有一个线程能执行
counter++;
unlock(&m);
```

### 28.2 Pthread 锁

```c
pthread_mutex_t lock = PTHREAD_MUTEX_INITIALIZER;

pthread_mutex_lock(&lock);
// 临界区
pthread_mutex_unlock(&lock);
```

### 28.3 实现一个锁

**评价标准**：

| 指标 | 说明 |
|------|------|
| 正确性 | 真正实现互斥 |
| 公平性 | 避免饥饿 |
| 性能 | 开销尽可能小 |

### 28.4 控制中断

**方案**：在临界区禁用中断

```c
void lock() {
    disable_interrupts();
}

void unlock() {
    enable_interrupts();
}
```

**问题**：
- 需要特权操作
- 可能导致系统无响应
- 多处理器无效

### 28.5 测试并设置指令（原子交换）

**硬件原语**：

```c
// 原子操作
int test_and_set(int *old, int new) {
    int prev = *old;
    *old = new;
    return prev;
}
```

### 28.6 实现可用的自旋锁

```c
typedef struct lock {
    int flag = 0;
} lock_t;

void lock(lock_t *l) {
    while (test_and_set(&l->flag, 1) == 1)
        ; // 自旋等待
}

void unlock(lock_t *l) {
    l->flag = 0;
}
```

### 28.7 评价自旋锁

| 方面 | 评价 |
|------|------|
| 正确性 | ✓ |
| 公平性 | ✗ 可能饥饿 |
| 性能 | 单CPU差，多CPU可接受 |

### 28.8 比较并交换

```c
int compare_and_swap(int *ptr, int expected, int new) {
    int actual = *ptr;
    if (actual == expected)
        *ptr = new;
    return actual;
}
```

### 28.9 链接的加载和条件式存储指令

**LL/SC (Load-Linked / Store-Conditional)**：

```c
int load_linked(int *ptr);
int store_conditional(int *ptr, int value);

void lock(lock_t *l) {
    while (1) {
        while (load_linked(&l->flag) == 1)
            ; // 等待
        if (store_conditional(&l->flag, 1) == 1)
            return; // 成功获取锁
    }
}
```

### 28.10 获取并增加

```c
int fetch_and_add(int *ptr) {
    int old = *ptr;
    *ptr = old + 1;
    return old;
}

// 可实现公平的票锁
```

### 28.11 自旋过多：怎么办

**问题**：自旋浪费 CPU

**解决方案**：让出 CPU

```c
void lock(lock_t *l) {
    while (test_and_set(&l->flag, 1) == 1)
        yield(); // 让出 CPU
}
```

### 28.12 使用队列：休眠替代自旋

**Linux futex 方案**：

```c
void lock(lock_t *l) {
    if (test_and_set(&l->flag, 1) == 1) {
        // 获取失败，加入等待队列
        futex_wait(&l->flag, 1);
    }
}

void unlock(lock_t *l) {
    l->flag = 0;
    // 唤醒等待线程
    futex_wake(&l->flag);
}
```

### 28.13 两阶段锁

**策略**：
1. 第一阶段：自旋一段时间
2. 第二阶段：如果未获取，休眠等待

### 28.14 小结

| 锁类型 | 适用场景 |
|--------|----------|
| 自旋锁 | 短临界区，多核 |
| 休眠锁 | 长临界区，单核 |
| 两阶段锁 | 通用 |
