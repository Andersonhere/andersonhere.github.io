## 问题4：Linux如何保证同宿主机上多个虚拟网络环境中路由表等可以独立工作？

### 原理

每个进程内部都有指针指向自己的网络命名空间：

```c
// file: include/linux/sched.h
struct task_struct {
    ...
    struct nsproxy *nsproxy;  // 命名空间代理
    ...
};

// file: include/linux/nsproxy.h
struct nsproxy {
    ...
    struct net *net_ns;  // 网络命名空间
    ...
};
```

数据包处理时根据 `skb->dev->nd_net` 确定命名空间。

## 10.2.2 veth底层创建过程

### 内核数据结构

```c
// file: drivers/net/veth.c
struct veth_priv {
    struct net_device __rcu *peer;  // 指向对端设备
    atomic64_t dropped;
};
```

### 创建流程

```c
// file: drivers/net/veth.c
static int veth_newlink(struct net *src_net, struct net_device *dev,
                        struct nlattr *tb[], struct nlattr *data[])
{
    // 1. 分配第一个net_device
    dev = rtnl_create_link(src_net, ifname, name_assign_type,
                           &veth_link_ops, tb);

    // 2. 分配第二个net_device（peer）
    peer = rtnl_create_link(net, ifname_peer, name_assign_type,
                            &veth_link_ops, tb_peer);

    // 3. 注册两个设备
    err = register_netdevice(dev);
    err = register_netdevice(peer);

    // 4. 互相设置peer指针
    priv = netdev_priv(dev);
    rcu_assign_pointer(priv->peer, peer);

    priv = netdev_priv(peer);
    rcu_assign_pointer(priv->peer, dev);

    return 0;
}
```

## 10.2.3 veth网络通信过程

### 发送函数

```c
// file: drivers/net/veth.c
static netdev_tx_t veth_xmit(struct sk_buff *skb, struct net_device *dev)
{
    struct veth_priv *priv = netdev_priv(dev);
    struct net_device *rcv;
    int length = skb->len;

    // 获取对端设备
    rcv = rcu_dereference(priv->peer);
    if (unlikely(!rcv))
        goto drop;

    // 将skb的dev改为对端设备
    skb->dev = rcv;

    // 将skb送入对端的接收队列
    if (likely(dev_forward_skb(rcv, skb) == NET_RX_SUCCESS)) {
        // 更新统计
        return NETDEV_TX_OK;
    }

drop:
    atomic64_inc(&priv->dropped);
    kfree_skb(skb);
    return NETDEV_TX_OK;
}
```

### dev_forward_skb函数

```c
// file: net/core/dev.c
int dev_forward_skb(struct net_device *dev, struct sk_buff *skb)
{
    // 设置skb所属的网络命名空间
    skb_scrub_packet(skb, true);

    // 设置skb的入口设备
    skb->dev = dev;

    // 将skb送入协议栈
    return netif_rx(skb);
}
```

## 10.3 网络命名空间

## 概述

网络命名空间（Network Namespace）是Linux实现网络隔离的核心技术，本节深入分析其工作原理。

## 10.3.1 如何使用网络命名空间

```bash
## 创建网络命名空间
ip netns add ns1

## 列出所有命名空间
ip netns list

## 在命名空间内执行命令
ip netns exec ns1 ip addr

## 删除命名空间
ip netns delete ns1
```

### 配置命名空间网络

```bash
## 创建veth对
ip link add veth0 type veth peer name veth1

ip netns exec ns1 ip link set lo up

## 在宿主机端配置IP
ip addr add 10.0.0.1/24 dev veth0
ip link set veth0 up

## 测试连通性
ip netns exec ns1 ping 10.0.0.1
```

### 命名空间内执行交互式命令


```c
struct net *copy_net_ns(unsigned long flags,
                        struct user_namespace *user_ns, struct net *old_net)

    struct net *net;
    int rv;

    if (!(flags & CLONE_NEWNET))
        return get_net(old_net);

}
```

## 10.4.3 添加设备

### 添加端口流程

```c
// file: net/bridge/br_if.c
int br_add_if(struct net_bridge *br, struct net_device *dev)
{
    struct net_bridge_port *p;

    // 1. 检查设备是否可以加入网桥
    if (dev->flags & IFF_LOOPBACK ||
        dev->type != ARPHRD_ETHER)
        return -EINVAL;

    // 2. 创建网桥端口
    p = new_nbp(br, dev);
    if (!p)
        return -ENOMEM;

    // 3. 设置设备的rx_handler
    err = netdev_rx_handler_register(dev, br_handle_frame, p);
    if (err)
        goto err1;

    // 4. 添加到端口列表
    list_add_rcu(&p->list, &br->port_list);

    // 5. 设置设备为混杂模式
    dev_set_promiscuity(dev, 1);

    return 0;
}
```

### 核心函数

```c
// file: net/bridge/br_input.c
rx_handler_result_t br_handle_frame(struct sk_buff **pskb)
{
    struct net_bridge_port *p;
    struct sk_buff *skb = *pskb;

    p = br_port_get_rcu(skb->dev);

    // 转发处理
    return br_handle_frame_finish(p->br, skb);
}

int br_handle_frame_finish(struct net_bridge *br, struct sk_buff *skb)
{
    // 1. 更新FDB（学习）
    br_fdb_update(br, p, eth_hdr(skb)->h_source, vid);

    // 2. 查找FDB
    dst = br_fdb_find(br, dest, vid);

    // 3. 转发
    if (dst) {
        br_forward(dst->dst, skb, skb2);
    } else {
        br_flood_forward(br, skb, skb2);  // 广播
    }
}
```

```
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  数据发送：                                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 本地进程 → OUTPUT → 路由判断 → POSTROUTING → 发出   │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  数据转发：                                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ PREROUTING → 路由判断 → FORWARD → POSTROUTING → 发出│   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  NAT类型：                                                  │
│  - SNAT：修改源IP（POSTROUTING链）                         │
│  - DNAT：修改目的IP（PREROUTING链）                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 10.5.2 实现外部网络通信

### 实验环境搭建


### 容器访问外部网络

```bash
## 1. 添加默认路由
ip netns exec net1 route add default gw 192.168.0.1 veth1

sysctl -w net.ipv4.conf.all.forwarding=1
iptables -P FORWARD ACCEPT

## 3. 配置SNAT（MASQUERADE）
iptables -t nat -A POSTROUTING -s 192.168.0.0/24 ! -o br0 -j MASQUERADE

## 4. 测试
```

## 10.6.4 问题解答

### 问题1：容器中的eth0和母机上的eth0是一个东西吗？

**答案**：不是。

- 母机的eth0是真正的物理网卡
- 容器的eth0是veth设备对的一端，是虚拟设备
- 名字可以随意改，命名成eth0只是让容器更像物理机

### 问题2：veth设备是什么，它是如何工作的？

**答案**：veth是一对虚拟网卡。

- 创建时同时创建两个设备，互相指向对方
- 一端发送的数据从另一端接收
- 不经过物理网络

### 问题3：Linux如何实现虚拟网络环境？

**答案**：通过网络命名空间。

- 每个命名空间有独立的网络栈
- 包括设备、IP、路由、iptables等
- 容器通过命名空间实现网络隔离

### 问题4：同一宿主机上多个容器如何通信？

**答案**：通过Bridge（网桥）。

- Bridge模拟交换机功能
- 根据MAC地址智能转发
- 工作在二层

### 问题5：容器如何和外部网络通信？

**答案**：通过路由和NAT。

- SNAT：容器访问外网时转换源IP
- DNAT：外部访问容器时转换目的IP

- 独立的网络设备
- 独立的IP地址
- 独立的路由表
- 独立的iptables

### 10.3.4 网络收发如何使用网络命名空间

数据包处理时根据 `skb->dev->nd_net` 确定命名空间。

## 10.4 虚拟交换机Bridge

### 10.4.1 如何使用Bridge

Bridge（网桥）类似于虚拟交换机：

```
┌──────────────────────────────────────┐
│            docker0 (Bridge)          │
│    ┌──────────┬──────────┬───────┐   │
│    │  veth0   │  veth2   │ veth4 │   │
└────┴──────────┴──────────┴───────┴───┘
     │          │          │
   容器1      容器2       容器3
```

**创建网桥**：

```bash
## 创建网桥
ip link add br0 type bridge
ip link set br0 up

## 将设备加入网桥
ip link set veth0 master br0
```

### 10.4.2 Bridge是如何创建出来的

```c
// file: net/bridge/br_device.c
static int br_dev_init(struct net_device *dev)
{
    struct net_bridge *br = netdev_priv(dev);

    // 创建网桥设备
    // 初始化FDB (转发数据库)
    br->hash_elasticity = BR_HASH_ELASTICITY;
    
    return 0;
}
```

### 10.4.3 添加设备

设备加入网桥后：
- 不能直接配置IP
- 由网桥统一转发
- 类似物理交换机端口

### 10.4.4 数据包处理过程

1. 数据到达网桥端口
2. 查询FDB表
3. 转发到目标端口
4. 或广播到所有端口

```
┌─────────────────────────────────────────────────────────────┐
│                    Bridge数据包处理                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  数据包到达                                                  │
│      │                                                      │
│      ▼                                                      │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  查询FDB表（MAC地址转发表）                           │   │
│  │                                                     │   │
│  │  ┌───────────────────────────────────────────────┐ │   │
│  │  │ MAC地址        │ 端口                          │ │   │
│  │  ├───────────────┼───────────────────────────────┤ │   │
│  │  │ aa:bb:cc:...  │ veth0                         │ │   │
│  │  │ dd:ee:ff:...  │ veth2                         │ │   │
│  │  └───────────────┴───────────────────────────────┘ │   │
│  └─────────────────────────────────────────────────────┘   │
│      │                                                      │
│      ├──── 找到 ────► 转发到目标端口                        │
│      │                                                      │
│      └──── 未找到 ────► 广播到所有端口                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 10.5 外部网络通信

### 10.5.1 路由和NAT

容器访问外部网络需要：
1. **路由**：配置默认网关
2. **NAT**：地址转换（MASQUERADE）

```bash
## 设置默认网关
ip route add default via 172.17.0.1

## NAT规则
iptables -t nat -A POSTROUTING -s 172.17.0.0/16 ! -o docker0 -j MASQUERADE
```

### 10.5.2 实现外部网络通信

**完整流程**：

```
容器 (10.0.0.2)
    ↓ veth
docker0 (Bridge)
    ↓ eth0 (宿主机)
NAT (MASQUERADE)
    ↓
外部网络
```

**端口映射**：

```bash
## 将容器端口映射到宿主机
docker run -p 8080:80 nginx

## 对应iptables规则
iptables -t nat -A PREROUTING -p tcp --dport 8080 -j DNAT --to-destination 10.0.0.2:80
```
