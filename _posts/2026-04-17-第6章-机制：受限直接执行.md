> **核心问题**：如何高效、安全地执行进程？

## 本章大纲

### 6.1 基本技巧：受限直接执行

**目标**：让进程直接在 CPU 上执行，同时保持控制权

```
┌─────────────────────────────────────────────┐
│              用户模式 (User Mode)            │
│  - 有限的指令权限                            │
│  - 不能直接访问硬件                          │
│  - 不能访问受保护的内存                      │
└─────────────────────────────────────────────┘
         ↕ (系统调用/陷阱)
┌─────────────────────────────────────────────┐
│              内核模式 (Kernel Mode)          │
│  - 完全的硬件访问权限                        │
│  - 可以执行特权指令                          │
│  - 管理系统资源                              │
└─────────────────────────────────────────────┘
```

### 6.2 问题 1：受限制的操作

**解决方案**：系统调用 (System Call)

```
用户进程                    内核
   │                         │
   ├── 陷阱指令 ────────────→│  进入内核模式
   │                         │
   │                         ├── 检查参数
   │                         │
   │                         ├── 执行操作
   │                         │
   │←── 陷阱返回 ────────────┤  返回用户模式
   │                         │
```

**关键机制**：
- **陷阱表 (Trap Table)**：内核启动时设置
- **陷阱指令**：用户程序请求内核服务
- **陷阱返回**：内核返回用户模式

### 6.3 问题 2：在进程之间切换

#### 协作式调度
- 进程主动让出 CPU（通过系统调用）
- 问题：恶意进程可能占用 CPU

#### 非协作式调度：时钟中断

```
进程 A              内核               进程 B
  │                  │                  │
  │←── 时钟中断 ─────┤                  │
  │                  │                  │
  │    保存 A 的状态  │                  │
  │                  │                  │
  │                  ├── 恢复 B 的状态 ──→│
  │                  │                  │
  │                  │←── 时钟中断 ──────┤
  │                  │                  │
  │←─ 恢复 A 的状态 ─┤                  │
  │                  │                  │
```

**上下文切换步骤**：
1. 时钟中断触发
2. 保存当前进程的寄存器状态
3. 调用调度器选择下一个进程
4. 恢复下一个进程的寄存器状态
5. 返回用户模式

### 6.4 并发问题

**中断处理中的并发**：
- 中断处理时可能发生其他中断
- 需要禁用中断或使用锁保护

**解决方案**：
- 中断屏蔽
- 锁机制
- 原子操作

### 6.5 小结

受限直接执行的关键机制：

| 机制 | 目的 |
|------|------|
| 用户/内核模式 | 保护系统安全 |
| 系统调用 | 受控访问硬件 |
| 陷阱表 | 定义异常处理入口 |
| 时钟中断 | 定期获得 CPU 控制权 |
| 上下文切换 | 切换运行中的进程 |

## 执行流程

```
┌────────────────────────────────────────────────────┐
│                    启动时                          │
│  内核初始化陷阱表，设置中断处理程序               │
└────────────────────────────────────────────────────┘
                        ↓
┌────────────────────────────────────────────────────┐
│                    运行进程                        │
│  用户模式执行，遇到系统调用时陷入内核             │
└────────────────────────────────────────────────────┘
                        ↓
┌────────────────────────────────────────────────────┐
│                    系统调用                        │
│  通过陷阱进入内核模式，执行后返回用户模式         │
└────────────────────────────────────────────────────┘
                        ↓
┌────────────────────────────────────────────────────┐
│                    进程切换                        │
│  时钟中断 → 保存状态 → 调度 → 恢复状态            │
└────────────────────────────────────────────────────┘
```
