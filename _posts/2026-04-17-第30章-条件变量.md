> **核心问题**：线程如何等待某个条件？

## 本章大纲

### 30.1 定义和程序

**条件变量**：允许线程等待特定条件

```c
pthread_cond_t cond = PTHREAD_COND_INITIALIZER;
pthread_mutex_t lock = PTHREAD_MUTEX_INITIALIZER;

// 等待条件
pthread_mutex_lock(&lock);
while (条件不满足) {
    pthread_cond_wait(&cond, &lock);
}
// 条件满足
pthread_mutex_unlock(&lock);

// 通知条件变化
pthread_cond_signal(&cond);  // 唤醒一个
pthread_cond_broadcast(&cond); // 唤醒所有
```

### 30.2 生产者/消费者问题

**问题描述**：
- 生产者向缓冲区放数据
- 消费者从缓冲区取数据
- 缓冲区有大小限制

```c
#define MAX 100

int buffer[MAX];
int fill = 0, use = 0, count = 0;

pthread_mutex_t lock = PTHREAD_MUTEX_INITIALIZER;
pthread_cond_t not_empty = PTHREAD_COND_INITIALIZER;
pthread_cond_t not_full = PTHREAD_COND_INITIALIZER;

void put(int value) {
    pthread_mutex_lock(&lock);
    while (count == MAX) {
        pthread_cond_wait(&not_full, &lock);
    }
    buffer[fill] = value;
    fill = (fill + 1) % MAX;
    count++;
    pthread_cond_signal(&not_empty);
    pthread_mutex_unlock(&lock);
}

int get() {
    pthread_mutex_lock(&lock);
    while (count == 0) {
        pthread_cond_wait(&not_empty, &lock);
    }
    int value = buffer[use];
    use = (use + 1) % MAX;
    count--;
    pthread_cond_signal(&not_full);
    pthread_mutex_unlock(&lock);
    return value;
}
```

### 30.3 覆盖条件

**问题**：`pthread_cond_signal` 可能唤醒错误的线程

**解决方案**：使用 `pthread_cond_broadcast`

```c
// 更安全但开销更大
pthread_cond_broadcast(&cond);
```

### 30.4 小结

**条件变量使用要点**：

| 要点 | 说明 |
|------|------|
| 必须配合锁 | 防止竞争条件 |
| 用 while 检查条件 | 防止虚假唤醒 |
| signal/broadcast 选择 | 单线程用 signal，多线程用 broadcast |

## 条件变量流程

```
消费者等待:
┌─────────────────────────────────────┐
│ lock(&lock)                         │
│ while (count == 0)                  │
│   cond_wait(&cond, &lock)           │←─┐
│ // 条件满足，继续执行                 │  │
│ unlock(&lock)                       │  │
└─────────────────────────────────────┘  │
                                         │
生产者通知:                               │
┌─────────────────────────────────────┐  │
│ lock(&lock)                         │  │
│ put(value)                          │  │
│ cond_signal(&cond) ─────────────────┼──┘
│ unlock(&lock)                       │
└─────────────────────────────────────┘
```
