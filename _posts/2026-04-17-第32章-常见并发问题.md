> **核心问题**：并发编程中常见的错误有哪些？

## 本章大纲

### 32.1 有哪些类型的缺陷

| 类型 | 占比 |
|------|------|
| 非死锁缺陷 | 97% |
| 死锁缺陷 | 3% |

### 32.2 非死锁缺陷

#### 违反原子性

```c
// 错误
if (thd->proc_info) {
    fputs(thd->proc_info, ...);  // 可能被其他线程修改
}

// 正确
pthread_mutex_lock(&lock);
if (thd->proc_info) {
    fputs(thd->proc_info, ...);
}
pthread_mutex_unlock(&lock);
```

#### 违反顺序

```c
// 错误：可能在线程初始化前就使用
Thread t = get_thread(id);
use_thread(t);  // t 可能未初始化

// 正确：确保初始化完成
pthread_mutex_lock(&lock);
Thread t = get_thread(id);
while (!t->initialized) {
    pthread_cond_wait(&cond, &lock);
}
use_thread(t);
pthread_mutex_unlock(&lock);
```

### 32.3 死锁缺陷

**死锁条件**（必须同时满足）：

1. 互斥
2. 持有并等待
3. 不可抢占
4. 循环等待

```
线程 A                 线程 B
  │                      │
lock(L1)               lock(L2)
  │                      │
lock(L2) ← 等待 ──── lock(L1) ← 等待
  │                      │
死锁！
```

#### 死锁预防

| 策略 | 说明 |
|------|------|
| 一次获取所有锁 | 破坏持有并等待 |
| 按顺序获取锁 | 破坏循环等待 |
| 抢占式锁 | 破坏不可抢占 |

#### 按顺序获取锁

```c
// 危险：可能导致死锁
void transfer(Account *a, Account *b, int amount) {
    lock(a);
    lock(b);
    a->balance -= amount;
    b->balance += amount;
    unlock(b);
    unlock(a);
}

// 安全：按地址顺序获取锁
void transfer(Account *a, Account *b, int amount) {
    Account *first = (a < b) ? a : b;
    Account *second = (a < b) ? b : a;
    lock(first);
    lock(second);
    // ...
}
```

### 32.4 小结

**避免并发问题**：

| 方法 | 说明 |
|------|------|
| 正确使用锁 | 保护共享数据 |
| 遵循顺序 | 避免死锁 |
| 使用工具 | 静态分析、动态检测 |

## 死锁检测

```
资源分配图:
┌────────┐     ┌────────┐
│ 线程 A │ ──→ │ 锁 L1  │
└────────┘     └────────┘
    ↓              ↑
┌────────┐     ┌────────┐
│ 锁 L2  │ ←── │ 线程 B │
└────────┘     └────────┘

如果图中有环，则存在死锁
```
