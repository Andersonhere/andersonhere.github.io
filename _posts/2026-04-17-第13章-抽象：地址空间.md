> **核心问题**：如何为程序提供独立的内存空间？

## 本章大纲

### 13.1 早期系统

**早期内存模型**：
- 单个程序独占物理内存
- 不需要虚拟化

### 13.2 多道程序和时分共享

**问题出现**：
- 多个程序需要同时驻留内存
- 需要保护进程之间不互相干扰

### 13.3 地址空间

**定义**：进程可以访问的所有内存地址的集合

**典型布局**：

```
虚拟地址空间
高地址 ┌──────────────────┐
       │      栈 ↓        │  局部变量、函数调用
       │       ...        │
       │                  │
       │       堆 ↑       │  动态分配
       ├──────────────────┤
       │    数据段        │  全局变量、静态变量
       ├──────────────────┤
       │    代码段        │  程序指令
低地址 └──────────────────┘
```

### 13.4 目标

| 目标 | 说明 |
|------|------|
| 透明 | 程序无需知道虚拟化 |
| 效率 | 尽量减少开销 |
| 保护 | 进程间隔离 |

### 13.5 小结

地址空间是内存虚拟化的核心抽象：
- 每个进程拥有独立的地址空间
- 操作系统负责地址转换
- 硬件提供支持

## 思考题

2. 栈和堆为什么相向增长？
