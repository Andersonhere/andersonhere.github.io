> **核心问题**：如何选择下一个运行的进程？

## 本章大纲

### 7.1 工作负载假设

**简化假设**：
1. 每个工作运行时间相同
2. 同时到达
3. 不涉及 I/O
4. 运行时间已知

### 7.2 调度指标

| 指标 | 公式 | 说明 |
|------|------|------|
| 周转时间 | T完成 - T到达 | 任务从到达到完成的总时间 |
| 响应时间 | T首次运行 - T到达 | 任务到首次运行的时间 |

### 7.3 先进先出 (FIFO)

**优点**：
- 简单易实现

**缺点**：
- 护航效应：短任务被长任务阻塞
- 周转时间可能很长

```
任务 A (100ms) ────────────────────────────────────┐
任务 B (10ms)                                       ├────────────
任务 C (10ms)                                       ├────────────

FIFO: A 先运行 100ms，B 和 C 等待
周转时间: A=100, B=110, C=120, 平均=110ms
```

### 7.4 最短任务优先 (SJF)

**策略**：优先运行最短的任务

```
任务 A (100ms) ────────────────────────────────────┐
任务 B (10ms)                    ├────────────
任务 C (10ms)                    ├────────────

SJF: B 先运行，然后 C，最后 A
周转时间: B=10, C=20, A=120, 平均=50ms ✓
```

**问题**：非抢占式，无法处理任务到达时间不同的情况

### 7.5 最短完成时间优先 (STCF)

**策略**：抢占式 SJF，新任务到达时比较剩余时间

```
时间 0: A 到达 (100ms)
时间 10: B 到达 (10ms), A 剩余 90ms
        → 抢占 A，运行 B
时间 20: C 到达 (10ms), B 剩余 0ms
        → B 完成，运行 C
时间 30: C 完成，恢复 A

周转时间: B=10, C=20, A=120, 平均=50ms ✓
```

### 7.6 新度量指标：响应时间

**问题**：STCF 对交互式任务不友好

```
响应时间 = 首次运行时间 - 到达时间
```

对于交互式系统，响应时间比周转时间更重要。

### 7.7 轮转调度 (Round Robin)

**策略**：
- 每个任务运行固定时间片
- 时间片用完后切换到下一个任务

```
时间片 = 1ms

任务 A: ├─├─├─├─├─├─├─├─├─├─
任务 B: └─├─├─├─├─├─├─├─├─├─
任务 C: └┬─┴─┴─┴─┴─┴─┴─┴─┴─┴─

响应时间改善，但周转时间变长
```

**时间片选择**：
- 太短：频繁切换，开销大
- 太长：退化为 FIFO
- 权衡：通常 10-100ms

### 7.8 结合 I/O

**策略**：
- 进程发起 I/O 时阻塞
- I/O 完成时唤醒进程
- 调度器可以在 I/O 期间运行其他进程

```
进程 A: 运行 ─ I/O ─ 运行 ─ I/O ─ 运行
进程 B:      └─ 运行 ─ 运行 ─┘

重叠 I/O 和计算，提高 CPU 利用率
```

### 7.9 无法预知

**现实问题**：不知道任务的运行时间

**解决方案**：
- 历史信息预测未来
- 多级反馈队列 (MLFQ)

### 7.10 小结

| 算法 | 抢占 | 优化目标 | 问题 |
|------|------|----------|------|
| FIFO | 否 | 简单 | 护航效应 |
| SJF | 否 | 周转时间 | 需预知时间 |
| STCF | 是 | 周转时间 | 响应时间长 |
| RR | 是 | 响应时间 | 周转时间长 |

**权衡**：
- 周转时间 vs 响应时间
- 简单性 vs 公平性
- 需要信息 vs 可用信息
