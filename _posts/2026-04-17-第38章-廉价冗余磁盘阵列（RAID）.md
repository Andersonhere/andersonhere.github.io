> **核心问题**：如何提高磁盘性能和可靠性？

## 本章大纲

### 38.1 接口和 RAID 内部

**RAID (Redundant Array of Inexpensive Disks)**：
- 将多个磁盘组合成逻辑单元
- 对上层透明
- 提高性能和/或可靠性

### 38.2 故障模型

| 故障类型 | 说明 |
|----------|------|
| 故障停止 | 磁盘完全停止工作 |
| 故障静默 | 数据损坏但磁盘继续工作 |

### 38.3 如何评估 RAID

| 指标 | 说明 |
|------|------|
| 容量 | 有效存储容量 |
| 可靠性 | 能容忍多少磁盘故障 |
| 性能 | 吞吐量、延迟 |

### 38.4 RAID 0 级：条带化

```
数据: A B C D E F G H

磁盘0: A C E G
磁盘1: B D F H

优点: 最高性能
缺点: 无冗余，一个磁盘故障则数据全丢
```

**特点**：
- 容量：N × 单盘容量
- 可靠性：无
- 性能：N 倍单盘

### 38.5 RAID 1 级：镜像

```
数据: A B C D

磁盘0: A B C D
磁盘1: A B C D (镜像)

优点: 高可靠性，读性能好
缺点: 容量利用率低(50%)
```

**特点**：
- 容量：N/2 × 单盘容量
- 可靠性：可容忍 1 个磁盘故障
- 性能：读 N 倍，写与单盘相同

### 38.6 RAID 4 级：奇偶校验

```
数据: A B C D
奇偶: P = A⊕B⊕C⊕D

磁盘0: A C
磁盘1: B D
磁盘2: P(AB) P(CD)

优点: 容量利用率高
缺点: 奇偶盘成为瓶颈
```

**特点**：
- 容量：(N-1) × 单盘容量
- 可靠性：可容忍 1 个磁盘故障
- 写性能：受奇偶盘限制

### 38.7 RAID 5 级：旋转奇偶校验

```
数据: A B C D E F G H I

磁盘0: A D G P(CFH)
磁盘1: B E H P(ADI)
磁盘2: C F I P(BEG)

奇偶信息分布在不同磁盘
```

**特点**：
- 容量：(N-1) × 单盘容量
- 可靠性：可容忍 1 个磁盘故障
- 性能：无奇偶盘瓶颈

### 38.8 RAID 比较：总结

| RAID 级别 | 容量 | 可靠性 | 读性能 | 写性能 |
|-----------|------|--------|--------|--------|
| RAID 0 | 100% | 无 | 高 | 高 |
| RAID 1 | 50% | 高 | 高 | 中 |
| RAID 4 | (N-1)/N | 中 | 高 | 低 |
| RAID 5 | (N-1)/N | 中 | 高 | 中 |
| RAID 6 | (N-2)/N | 最高 | 高 | 低 |

### 38.9 其他有趣的 RAID 问题

**重建**：故障磁盘替换后恢复数据

**RAID 6**：双重奇偶校验，可容忍 2 个磁盘故障

### 38.10 小结

**RAID 选择指南**：

| 需求 | 推荐 |
|------|------|
| 最高性能 | RAID 0 |
| 高可靠性 | RAID 1 |
| 平衡性能与容量 | RAID 5 |
| 最高可靠性 | RAID 6 |
