> **核心问题**：如何管理空闲内存块？

## 本章大纲

### 17.1 假设

- 内存分配大小任意
- 内存请求大小各异
- 已分配内存不能移动

### 17.2 底层机制

#### 分割与合并

```
空闲列表: [10KB] [20KB] [15KB]

请求 12KB:
  从 20KB 分割 → [10KB] [8KB] [15KB]

释放相邻块:
  [10KB] + [8KB] → [18KB] (如果相邻)
```

#### 跟踪已分配块

**方式**：在分配块头部存储元数据

```
已分配块结构:
┌─────────────────────────────┐
│ 头部: 大小 + 魔数(校验)      │
├─────────────────────────────┤
│       用户数据区             │
│         ...                 │
└─────────────────────────────┘
```

### 17.3 基本策略

#### 最优适配 (Best Fit)

```
策略: 找到能满足请求的最小块

优点: 浪费最少
缺点: 需要遍历所有块，产生小碎片
```

#### 最差适配 (Worst Fit)

```
策略: 找到最大的块

优点: 避免产生小碎片
缺点: 大块很快被用完
```

#### 首次适配 (First Fit)

```
策略: 找到第一个足够大的块

优点: 简单，速度快
缺点: 前端碎片积累
```

#### 下次适配 (Next Fit)

```
策略: 从上次分配位置继续查找

优点: 分散碎片
缺点: 可能比首次适配更差
```

### 17.4 其他方式

#### 分离空闲列表

**思想**：为常用大小维护独立列表

```
小对象列表 (≤64B):  ─→ [块][块][块]...
中对象列表 (≤1KB):  ─→ [块][块]...
大对象列表 (>1KB):  ─→ [块]...
```

#### 伙伴系统

**特点**：按 2 的幂次方分配

```
初始: [256KB]

请求 10KB → 分配 16KB:
  256KB → 128KB → 64KB → 32KB → 16KB

释放时合并:
  两个相邻的 16KB → 32KB
```

### 17.5 小结

| 策略 | 复杂度 | 碎片情况 |
|------|--------|----------|
| 最优适配 | O(n) | 小碎片多 |
| 最差适配 | O(n) | 大块消失快 |
| 首次适配 | O(n) | 前端碎片 |
| 下次适配 | O(n) | 分散碎片 |
| 分离列表 | O(1) | 较好 |
| 伙伴系统 | O(log n) | 内部碎片 |
