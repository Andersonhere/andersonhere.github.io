> **核心主题**：引入并发的概念

## 本章要点

### 什么是并发？

并发是指多个执行实体（线程/进程）同时存在并可能同时执行的能力。

### 为什么需要并发？

1. **并行计算**：充分利用多核 CPU
2. **I/O 重叠**：等待 I/O 时继续执行
3. **模块化设计**：将复杂任务分解为独立部分

### 并发带来的挑战

| 问题 | 说明 |
|------|------|
| 竞争条件 | 多线程访问共享数据 |
| 死锁 | 线程互相等待 |
| 活锁 | 线程持续但无进展 |
| 饥饿 | 某些线程无法获得资源 |

### 核心概念

```
单线程程序:
  ┌─────────────────────────────────┐
  │  执行流                          │
  └─────────────────────────────────┘

多线程程序:
  ┌─────────────┐
  │  线程 1      │
  └─────────────┘
        ↓ 共享数据
  ┌─────────────┐
  │  线程 2      │
  └─────────────┘
```
