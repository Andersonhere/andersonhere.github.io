> **核心问题**：如何实现虚拟地址到物理地址的转换？

## 本章大纲

### 15.1 假设

**简化假设**：
- 地址空间连续
- 地址空间小于物理内存
- 地址空间大小相同

### 15.2 一个例子

**动态重定位 (基址+界限)**：

```
虚拟地址        物理地址
   0      →     16KB + 0
   1KB    →     16KB + 1KB
   ...    →     ...

基址寄存器 = 16KB
界限寄存器 = 4KB

转换公式：物理地址 = 基址 + 虚拟地址
界限检查：虚拟地址 < 界限
```

### 15.3 动态（基于硬件）重定位

**硬件支持**：

| 组件 | 功能 |
|------|------|
| 基址寄存器 | 地址偏移 |
| 界限寄存器 | 边界检查 |
| CPU 特权模式 | 保护寄存器 |

**地址转换过程**：

```
CPU 生成虚拟地址
        ↓
    界限检查
        ↓ (通过)
物理地址 = 基址 + 虚拟地址
        ↓
    访问内存
```

### 15.4 硬件支持：总结

| 任务 | 责任方 |
|------|--------|
| 地址转换 | 硬件 (MMU) |
| 界限检查 | 硬件 |
| 异常处理 | 操作系统 |
| 寄存器管理 | 操作系统 |

### 15.5 操作系统的问题

**操作系统职责**：

1. **进程创建**：分配物理内存空间
2. **进程终止**：回收内存
3. **上下文切换**：保存/恢复基址界限寄存器
4. **异常处理**：处理越界访问

```
上下文切换：

保存旧进程:
  PCB->base = base_register
  PCB->limit = limit_register

恢复新进程:
  base_register = PCB->base
  limit_register = PCB->limit
```

### 15.6 小结

**动态重定位的优缺点**：

| 优点 | 缺点 |
|------|------|
| 简单高效 | 外部碎片 |
| 硬件支持少 | 内部碎片 |
| 保护性好 | 地址空间需连续 |
