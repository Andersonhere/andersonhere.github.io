## 一、Linux调度器发展历程

Linux 内核的调度器经历了多次重大演进：

| 时期 | 调度器 | 特点 |
|------|--------|------|
| Linux 2.4 | O(n)调度器 | 遍历所有进程选择最优，时间复杂度O(n) |
| Linux 2.6.0 | O(1)调度器 | 恒定时间完成调度，引入优先级数组 |
| Linux 2.6.23 | CFS | 完全公平调度，基于红黑树实现 |
| Linux 3.14 | 增加Deadline调度器 | 支持实时任务的截止时间调度 |

## 二、CFS（完全公平调度器）

### 2.1 设计理念

CFS（Completely Fair Scheduler）的核心思想是**公平性**：在理想状态下，每个任务都应该获得公平的CPU时间份额。

> **核心理念**：模型化一个"理想的多任务CPU"，该CPU拥有100%的物理算力，每个进程都能以精确的速度并行运行。

在现实中无法实现真正的并行，CFS 通过以下机制模拟公平性：

- **虚拟运行时间（vruntime）**：记录每个进程获得的CPU时间（经优先级加权）
- **红黑树**：维护所有可运行进程，按vruntime排序
- **总是选择vruntime最小的进程运行**：确保公平

### 2.2 虚拟运行时间

vruntime 是 CFS 的核心概念：

```
vruntime += delta_exec * (NICE_0_LOAD / weight)
```

其中：
- `delta_exec`：实际运行时间
- `NICE_0_LOAD`：标准权重（nice值为0的进程权重）
- `weight`：进程权重（由nice值决定）

**权重与nice值的对应关系**：

| nice值 | 权重 |
|--------|------|
| -5 | 3121 |
| -4 | 2611 |
| -3 | 2169 |
| -2 | 1763 |
| -1 | 1440 |
| 0 | 1024 |
| 1 | 820 |
| 2 | 655 |
| 3 | 510 |
| 4 | 383 |
| 5 | 261 |

nice值每降低1，进程获得约25%更多的CPU时间；每升高1，减少约25%。

### 2.3 调度周期与时间片

CFS 引入了"调度延迟"（sched_latency）概念：

```c
// 默认值为 6ms（可配置）
目标调度延迟 = sysctl_sched_latency

// 每个进程的时间片
时间片 = 目标调度延迟 * (进程权重 / 总权重)
```

当进程数量过多时，会引入"最小粒度"（min_granularity，默认0.75ms）保证时间片不会太小。

### 2.4 红黑树结构

CFS 使用红黑树组织所有可运行的进程：

```
                    [vruntime=20]
                   /             \
          [vruntime=10]     [vruntime=30]
             /      \
      [vruntime=5]  [vruntime=15]
```

- **最左侧节点**：vruntime最小，优先被调度
- **插入/删除**：O(log n) 时间复杂度
- **缓存最左侧节点**：避免每次遍历

### 2.5 进程唤醒与抢占

当进程从休眠状态唤醒时：

1. 如果vruntime过小，可能导致它长时间占用CPU
2. CFS通过`sched_vslice()`计算补偿vruntime
3. 通常设置为min_vruntime，防止进程饥饿

抢占条件：

```c
if (curr->vruntime - p->vruntime > sysctl_sched_wakeup_granularity)
    resched_task(curr);
```

## 三、进程优先级

### 3.1 静态优先级（nice值）

nice值范围：-20 到 +19，默认为0

```bash
# 启动时设置
nice -n 10 ./my_program

# 运行时修改
renice -n 5 -p <pid>
```

### 3.2 实时优先级

实时进程优先级范围：0-99（数值越大优先级越高）

实时进程类型：
- **SCHED_FIFO**：先进先出，不时间片，主动让出或被更高优先级抢占
- **SCHED_RR**：轮转调度，有时间片
- **SCHED_DEADLINE**：截止时间调度（Linux 3.14+）

```bash
# 查看实时优先级
chrt -p <pid>

# 设置为FIFO调度，优先级50
chrt -f -p 50 <pid>
```

### 3.3 优先级映射

Linux 将各种优先级统一到 0-139 的范围：

| 范围 | 类型 | 说明 |
|------|------|------|
| 0-99 | 实时进程 | 对应SCHED_FIFO/SCHED_RR |
| 100-139 | 普通进程 | nice值映射（100+nice+20） |

## 四、调度类

Linux 将调度器模块化为调度类（scheduling class）：

```c
struct sched_class {
    const struct sched_class *next;  // 链式结构

    void (*enqueue_task)(...);       // 入队
    void (*dequeue_task)(...);       // 出队
    void (*check_preempt_curr)(...); // 抢占检查
    struct task_struct *(*pick_next_task)(...); // 选择下一个任务
    // ...
};
```

调度类优先级（从高到低）：

1. **dl_sched_class**：Deadline调度类
2. **rt_sched_class**：实时调度类
3. **fair_sched_class**：CFS调度类
4. **idle_sched_class**：空闲调度类

调度时从高优先级调度类开始选择任务。

## 五、CPU亲和性与负载均衡

### 5.1 CPU亲和性

进程可以被绑定到特定CPU核心：

```bash
# 查看亲和性
taskset -p <pid>

# 设置亲和性（绑定到CPU 0和2）
taskset -p 0x5 <pid>
```

### 5.2 负载均衡

CFS 在每个CPU核心上维护独立的红黑树，通过以下机制实现负载均衡：

1. **周期性负载均衡**：定时检查各CPU负载
2. **空闲负载均衡**：CPU空闲时从繁忙CPU拉取任务
3. **主动负载均衡**：迁移高负载CPU上的任务

负载均衡考虑因素：
- 各CPU运行队列长度
- 缓存亲和性
- NUMA拓扑结构

### 5.3 调度域

Linux 使用调度域（scheduling domain）描述CPU拓扑：

```
系统级调度域
    ├── NUMA节点0
    │   ├── LLC共享域
    │   │   ├── CPU 0
    │   │   ├── CPU 1
    │   │   └── CPU 2
    │   └── ...
    └── NUMA节点1
        └── ...
```

负载均衡在不同层级有不同的策略，避免不必要的跨NUMA迁移。

## 七、CFS 与 MLFQ 对比

| 特性 | CFS | MLFQ |
|------|-----|------|
| 数据结构 | 红黑树 | 多级队列 |
| 公平性 | 基于vruntime严格公平 | 基于启发式规则 |
| 优先级处理 | 权重转换 | 队列层级
| I/O密集型 | 自动降低vruntime | 保持高优先级队列 |
| 时间复杂度 | O(log n) | O(1) |
| 配置复杂度 | 较低 | 较高（多层参数） |

## 八、实践案例

### 8.1 观察调度行为

```c
// sched_demo.c
#include <stdio.h>
#include <unistd.h>
#include <sched.h>

int main() {
    struct sched_param param;
    int policy = sched_getscheduler(0);

    printf("Current scheduling policy: ");
    switch(policy) {
        case SCHED_OTHER: printf("SCHED_OTHER (CFS)\n"); break;
        case SCHED_FIFO:  printf("SCHED_FIFO\n"); break;
        case SCHED_RR:    printf("SCHED_RR\n"); break;
    }

    printf("Nice value: %d\n", nice(0));
    printf("PID: %d\n", getpid());

    // 模拟CPU密集型任务
    while(1) {
        volatile int i;
        for(i = 0; i < 10000000; i++);
        printf("PID %d running\n", getpid());
    }
    return 0;
}
```

### 8.2 实时任务示例

```c
#include <sched.h>
#include <stdio.h>
#include <unistd.h>

int main() {
    struct sched_param param;
    param.sched_priority = 50;  // 实时优先级

    // 设置为SCHED_FIFO
    if (sched_setscheduler(0, SCHED_FIFO, &param) == -1) {
        perror("sched_setscheduler");
        return 1;
    }

    printf("Running as SCHED_FIFO with priority 50\n");

    // 需要root权限
    while(1) {
        // 实时任务逻辑
    }
    return 0;
}
```

## 九、总结

Linux CFS 调度器通过以下核心机制实现了公平高效的进程调度：

1. **虚拟运行时间**：量化每个进程获得的CPU时间
2. **红黑树**：高效组织可运行进程
3. **权重系统**：支持优先级差异
4. **模块化设计**：调度类支持多种调度策略

CFS 相比传统 MLFQ 更简单、更公平，成为现代 Linux 的默认调度器。理解 CFS 有助于优化应用程序性能和调试调度相关问题。
