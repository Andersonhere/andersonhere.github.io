> **核心问题**：如何加速地址转换？

## 本章大纲

### 19.1 TLB 的基本算法

**TLB (Translation Lookaside Buffer)**：地址转换缓存

```
虚拟地址 → 提取 VPN → TLB 查找
                     ↓
              ┌──────┴──────┐
              命中          未命中
              ↓             ↓
            返回 PFN    访问页表
                          ↓
                        更新 TLB
                          ↓
                        重试
```

### 19.2 示例：访问数组

```c
int arr[1000];
for (int i = 0; i < 1000; i++) {
    arr[i] = 0;  // 数组访问
}
```

**TLB 行为分析**：

```
第一次访问 arr[0]:
  TLB 未命中 → 访问页表 → 更新 TLB

后续访问 arr[1]...arr[n]:
  TLB 命中 (同一页内的访问)

跨页时:
  TLB 未命中 → 更新 TLB
```

**TLB 命中率**：取决于局部性

### 19.3 谁来处理 TLB 未命中

**两种方式**：

| 方式 | 责任方 | 特点 |
|------|--------|------|
| 硬件管理 | CPU | 快，但灵活性低 |
| 软件管理 | 操作系统 | 灵活，但需要陷入内核 |

**软件管理流程**：
1. TLB 未命中触发异常
2. 操作系统查找页表
3. 更新 TLB
4. 返回用户模式重试

### 19.4 TLB 的内容

**典型 TLB 表项**：

```
┌─────────────────────────────────────────────────┐
│ VPN │ PFN │ 有效 │ 保护 │ 脏 │ 全局 │ ASID │
└─────────────────────────────────────────────────┘

VPN: 虚拟页号
PFN: 物理页框号
有效: 表项是否有效
保护: 读/写/执行权限
脏: 页是否被修改
全局: 是否对所有进程有效
ASID: 地址空间标识符
```

### 19.5 上下文切换时对 TLB 的处理

**问题**：切换进程时 TLB 包含旧进程的转换

**解决方案**：

| 方案 | 说明 |
|------|------|
| 全部清空 | 简单但效率低 |
| ASID 标识 | 不同进程的转换共存 |

```
TLB (带 ASID):
┌──────┬──────┬──────┐
│ ASID │ VPN  │ PFN  │
├──────┼──────┼──────┤
│  1   │ 0x10 │ 0x25 │  进程 1
│  2   │ 0x10 │ 0x30 │  进程 2
└──────┴──────┴──────┘
```

### 19.6 TLB 替换策略

| 策略 | 说明 |
|------|------|
| LRU | 最近最少使用 |
| 随机 | 随机选择 |
| NRU | 近期未使用 |

### 19.7 实际系统的 TLB 表项

**x86-64 示例**：
- L1 TLB: 64 条目（数据）+ 128 条目（指令）
- L2 TLB: 1536 条目（统一）
- 4KB 页 + 2MB 大页支持

### 19.8 小结

**TLB 优化效果**：

| 情况 | 内存访问次数 |
|------|--------------|
| 无 TLB | 2 次 |
| TLB 命中 | 1 次 |
| TLB 未命中 | 2+ 次 |
