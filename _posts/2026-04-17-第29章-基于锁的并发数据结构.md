> **核心问题**：如何设计线程安全的数据结构？

## 本章大纲

### 29.1 并发计数器

**简单方案**：全局锁

```c
typedef struct {
    int value;
    pthread_mutex_t lock;
} counter_t;

void increment(counter_t *c) {
    pthread_mutex_lock(&c->lock);
    c->value++;
    pthread_mutex_unlock(&c->lock);
}
```

**问题**：锁竞争严重

**优化方案**：近似计数器

```
每个 CPU 一个局部计数器
定期同步到全局计数器

CPU 0: [局部计数] ──┐
CPU 1: [局部计数] ──┼──→ [全局计数]
CPU 2: [局部计数] ──┘
```

### 29.2 并发链表

**简单方案**：全链表一把锁

```c
void insert(list_t *l, int value) {
    pthread_mutex_lock(&l->lock);
    // 插入操作
    pthread_mutex_unlock(&l->lock);
}
```

**优化方案**：细粒度锁

```c
// 每个节点一个锁
typedef struct node {
    int value;
    struct node *next;
    pthread_mutex_t lock;
} node_t;
```

**手递锁 (Hand-over-hand locking)**：

```
遍历时:
  锁住节点 A → 移动到 B → 锁住 B → 解锁 A → ...
```

### 29.3 并发队列

**方案**：头尾分离锁

```c
typedef struct {
    node_t *head;
    node_t *tail;
    pthread_mutex_t head_lock;
    pthread_mutex_t tail_lock;
} queue_t;

void enqueue(queue_t *q, int value) {
    pthread_mutex_lock(&q->tail_lock);
    // 入队操作
    pthread_mutex_unlock(&q->tail_lock);
}

int dequeue(queue_t *q) {
    pthread_mutex_lock(&q->head_lock);
    // 出队操作
    pthread_mutex_unlock(&q->head_lock);
}
```

### 29.4 并发散列表

**方案**：每个桶一个锁

```c
typedef struct {
    list_t buckets[NBUCKETS];
    pthread_mutex_t locks[NBUCKETS];
} hashtable_t;

void insert(hashtable_t *h, int key, int value) {
    int bucket = hash(key) % NBUCKETS;
    pthread_mutex_lock(&h->locks[bucket]);
    // 插入操作
    pthread_mutex_unlock(&h->locks[bucket]);
}
```

### 29.5 小结

**设计原则**：

| 原则 | 说明 |
|------|------|
| 避免不必要共享 | 减少竞争 |
| 合理划分锁粒度 | 平衡开销与并行度 |
| 注意锁开销 | 细粒度锁增加管理成本 |

## 并发数据结构对比

| 结构 | 简单方案 | 优化方案 |
|------|----------|----------|
| 计数器 | 全局锁 | 近似计数 |
| 链表 | 全局锁 | 细粒度锁 |
| 队列 | 全局锁 | 头尾分离 |
| 散列表 | 全局锁 | 桶级锁 |
