> **核心问题**：如何创建和控制进程？

## 本章大纲

### 5.1 fork() 系统调用

```c
pid_t pid = fork();
if (pid == 0) {
    // 子进程
} else {
    // 父进程
}
```

**特点**：
- 调用一次，返回两次
- 子进程获得父进程的副本（代码、数据、栈、堆）
- 子进程从 fork() 返回处开始执行

### 5.2 wait() 系统调用

```c
pid_t wait(int *status);
pid_t waitpid(pid_t pid, int *status, int options);
```

**功能**：
- 父进程等待子进程结束
- 获取子进程退出状态
- 防止产生僵尸进程

### 5.3 exec() 系统调用

```c
int execl(const char *path, const char *arg, ...);
int execv(const char *path, char *const argv[]);
int execvp(const char *file, char *const argv[]);
```

**功能**：
- 替换当前进程的地址空间
- 加载并执行新程序
- 不会返回（除非出错）

**保留的属性**：
- 进程 ID (PID)
- 打开的文件描述符
- 工作目录
- 信号处理设置（大部分重置）

### 5.4 为什么这样设计 API

**分离 fork 和 exec 的好处**：

1. **灵活性**：fork 后可以修改子进程状态
2. **简单性**：fork 只做一件事——复制进程
3. **组合性**：fork + exec 组合实现进程创建

```c
// 典型模式
if (fork() == 0) {
    // 子进程：重定向、设置环境等
    close(STDOUT_FILENO);
    open("output.txt", O_WRONLY);
    execvp("ls", argv);
}
wait(NULL);  // 父进程等待
```

### 5.5 其他 API

| API | 功能 |
|-----|------|
| `getpid()` | 获取当前进程 ID |
| `getppid()` | 获取父进程 ID |
| `exit(int status)` | 进程退出 |
| `kill(pid_t pid, int sig)` | 发送信号 |
| `signal(int sig, handler)` | 设置信号处理函数 |

### 5.6 小结

UNIX 进程 API 设计哲学：
- **简单原语**：每个函数做一件事
- **可组合**：通过组合实现复杂功能
- **一致性**：统一的设计风格

## 进程创建流程图

```
父进程                    子进程
  │
  ├─── fork() ───────────→ 复制父进程
  │                        │
  │                        ├── 返回 0
  │                        │
  │   返回子进程 PID        │
  │←───────────────────────┤
  │                        │
  │                        ├── exec() 加载新程序
  │                        │
  ├─── wait() ─────────────┤
  │                        │
  │                        └── exit() 退出
  │←───────────────────────┘
  │
  └── 继续执行
```
