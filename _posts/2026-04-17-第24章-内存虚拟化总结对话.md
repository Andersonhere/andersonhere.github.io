> **核心主题**：内存虚拟化知识总结

## 核心机制回顾

### 地址转换

| 技术 | 特点 |
|------|------|
| 基址+界限 | 简单，有碎片 |
| 分段 | 支持稀疏空间，外部碎片 |
| 分页 | 无外部碎片，页表大 |
| 多级页表 | 节省空间，增加延迟 |

### 加速技术

| 技术 | 作用 |
|------|------|
| TLB | 加速地址转换 |
| 多级缓存 | 减少内存访问 |
| 大页 | 减少 TLB 未命中 |

### 超越物理内存

| 技术 | 作用 |
|------|------|
| 交换空间 | 扩展可用内存 |
| 页替换 | 管理有限内存 |
| 预读 | 减少 I/O |

## 关键问题与答案

| 问题 | 答案 |
|------|------|
| 如何隔离进程？ | 独立地址空间 |
| 如何高效转换？ | TLB |
| 如何处理大地址空间？ | 多级页表 |
| 内存不足怎么办？ | 交换 + 页替换 |

## 最佳实践

1. **利用局部性**：减少 TLB 未命中
2. **选择合适的页大小**：平衡碎片和页表大小
3. **避免抖动**：控制工作集
