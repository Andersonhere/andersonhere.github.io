> **核心问题**：文件系统如何组织数据？

## 本章大纲

### 40.1 思考方式

**两个视角**：

| 视角 | 关注点 |
|------|--------|
| 用户视角 | 文件、目录、路径 |
| 系统视角 | 数据块、索引、元数据 |

### 40.2 整体组织

**磁盘布局**：

```
┌─────────┬──────────┬──────────┬────────────────┐
│ 超级块  │ inode区  │ 数据位图 │ 数据区         │
└─────────┴──────────┴──────────┴────────────────┘

超级块: 文件系统元信息
inode区: 文件元数据
数据位图: 数据块使用情况
数据区: 实际文件内容
```

### 40.3 文件组织：inode

**inode 结构**：

```c
struct inode {
    int size;           // 文件大小
    int type;           // 文件类型
    int permissions;    // 权限
    int direct[12];     // 直接指针
    int indirect;       // 一级间接指针
    int double_indirect; // 二级间接指针
};
```

**多级索引**：

```
文件数据访问:
- 小文件: 直接指针 (快速)
- 中等文件: 一级间接 (12 + 256 个块)
- 大文件: 二级间接 (更大容量)
```

### 40.4 目录组织

**目录结构**：

```c
struct dirent {
    int inode_num;      // inode 编号
    char name[256];     // 文件名
};
```

```
目录数据:
┌──────────────────────────────────────┐
│ inode: 10, name: "."                 │
│ inode: 2,  name: ".."                │
│ inode: 15, name: "file1.txt"         │
│ inode: 20, name: "subdir"            │
└──────────────────────────────────────┘
```

### 40.5 空闲空间管理

**位图法**：

```
数据块使用情况:
位图: 0 1 1 0 0 1 1 1 0 0 ...
       ↑           ↑
      空闲        已用
```

### 40.6 访问路径：读取和写入

#### 读取文件

```
打开 /foo/bar:
1. 读取根目录 inode
2. 在根目录数据中找 foo
3. 读取 foo 的 inode
4. 在 foo 数据中找 bar
5. 读取 bar 的 inode

读取 bar 的内容:
6. 根据 inode 中的指针读取数据块
```

#### 写入文件

```
写入过程:
1. 定位文件 inode
2. 分配新数据块 (更新位图)
3. 写入数据
4. 更新 inode (大小、时间)
5. 写回 inode
```

### 40.7 缓存和缓冲

**页缓存**：

```
┌─────────────────────────────────────────┐
│              内存缓存                    │
│  ┌─────┬─────┬─────┬─────┬─────┐      │
│  │文件A│文件B│文件C│ ... │     │      │
│  │块1 │块2 │块1 │     │     │      │
│  └─────┴─────┴─────┴─────┴─────┘      │
└─────────────────────────────────────────┘
        ↑                ↑
      读缓存           写缓冲
```

**优点**：
- 减少磁盘访问
- 提高性能

**风险**：
- 崩溃可能丢失未写入数据

### 40.8 小结

| 组件 | 功能 |
|------|------|
| 超级块 | 文件系统信息 |
| inode | 文件元数据 |
| 数据位图 | 空间管理 |
| 数据块 | 文件内容 |
