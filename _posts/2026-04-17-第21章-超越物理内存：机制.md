> **核心问题**：地址空间大于物理内存时如何处理？

## 本章大纲

### 21.1 交换空间

**交换空间 (Swap Space)**：磁盘上用于存储换出页的区域

```
物理内存              交换空间
┌────────┐           ┌────────┐
│ 页框 0 │           │ 页 0   │
│ 页框 1 │ ←────────→│ 页 1   │
│ 页框 2 │           │ 页 2   │
│ ...    │           │ ...    │
└────────┘           └────────┘
```

### 21.2 存在位

**页表项增加存在位**：

```
┌─────────────────────────────────────┐
│ 存在位 │ 脏位 │ 保护位 │ PFN/磁盘地址 │
└─────────────────────────────────────┘

存在位 = 1: 页在物理内存中，PFN 有效
存在位 = 0: 页在磁盘中，磁盘地址有效
```

### 21.3 页错误

**页错误处理流程**：

```
访问虚拟地址
    ↓
页表查找
    ↓
存在位 = 0?
    ↓ 是
触发页错误异常
    ↓
操作系统处理:
  1. 在交换空间找到页
  2. 分配物理页框
  3. 从磁盘读取页
  4. 更新页表
  5. 重新执行指令
```

### 21.4 内存满了怎么办

**页替换**：选择一个页换出

```
物理内存满时:
  1. 选择牺牲页 (替换策略)
  2. 如果脏页，写回磁盘
  3. 更新页表 (存在位 = 0)
  4. 使用释放的页框
```

### 21.5 页错误处理流程

**详细流程**：

```
┌─────────────────────────────────────────────────┐
│ 1. CPU 访问虚拟地址                              │
│ 2. MMU 查找页表，存在位=0                        │
│ 3. 触发页错误异常                                │
│ 4. OS 保存当前进程状态                           │
│ 5. OS 查找页在磁盘的位置                         │
│ 6. OS 选择牺牲页，换出（如需要）                  │
│ 7. OS 从磁盘读取页到内存                         │
│ 8. OS 更新页表                                   │
│ 9. OS 返回用户模式，重试指令                     │
└─────────────────────────────────────────────────┘
```

### 21.6 交换何时真正发生

**交换守护进程 (Swap Daemon)**：

- 后台运行
- 定期检查内存使用
- 主动换出未使用的页
- 保持一定数量的空闲页

### 21.7 小结

**虚拟内存机制总结**：

| 组件 | 功能 |
|------|------|
| 存在位 | 判断页是否在内存 |
| 页错误 | 处理缺页 |
| 交换空间 | 存储换出的页 |
| 替换策略 | 选择牺牲页 |
