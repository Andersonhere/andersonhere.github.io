> **核心问题**：为什么并发很难？

## 本章大纲

### 26.1 实例：线程创建

```c
#include <pthread.h>
#include <stdio.h>

void *mythread(void *arg) {
    printf("%s\n", (char *)arg);
    return NULL;
}

int main() {
    pthread_t p1, p2;
    pthread_create(&p1, NULL, mythread, "A");
    pthread_create(&p2, NULL, mythread, "B");
    pthread_join(p1, NULL);
    pthread_join(p2, NULL);
    return 0;
}
```

**输出可能是**：
- A, B
- B, A
- 或交错输出

### 26.2 为什么更糟糕：共享数据

**问题代码**：

```c
int counter = 0;

void *increment(void *arg) {
    for (int i = 0; i < 10000; i++) {
        counter++;  // 非原子操作！
    }
    return NULL;
}

// 两个线程各增加 10000 次
// 结果可能 < 20000
```

### 26.3 核心问题：不可控的调度

**counter++ 的实际执行**：

```
counter++ 实际是三步:
1. LOAD: 将 counter 加载到寄存器
2. ADD:  寄存器值 +1
3. STORE: 将结果存回 counter

可能的交错执行:
线程 A                    线程 B
LOAD counter (0)
                          LOAD counter (0)
ADD (1)
                          ADD (1)
STORE counter (1)
                          STORE counter (1)
结果: counter = 1 (预期 2)
```

### 26.4 原子性愿望

**原子操作**：不可分割的操作

```c
// 期望: counter++ 是原子的
// 实际: 需要同步机制

// 解决方案: 使用锁
pthread_mutex_lock(&lock);
counter++;
pthread_mutex_unlock(&lock);
```

### 26.5 还有一个问题：等待另一个线程

**生产者-消费者问题**：

```c
// 消费者等待数据
while (buffer_empty()) {
    // 忙等待？浪费 CPU
    // 睡眠？可能错过信号
}
```

**解决方案**：条件变量

### 26.6 小结：为什么操作系统课要研究并发

| 原因 | 说明 |
|------|------|
| 内核并发 | 操作系统本身就是多线程 |
| 系统调用 | 用户程序需要同步机制 |
| 并行计算 | 多核时代必备技能 |

## 竞争条件示意图

```
时间轴:

线程 A: ──LOAD──ADD──STORE──
线程 B: ─────LOAD──ADD──STORE──

正确顺序 (串行):
A: LOAD(0)→ADD→STORE(1)
B: LOAD(1)→ADD→STORE(2)
结果: 2

错误顺序 (交错):
A: LOAD(0)
B:    LOAD(0)
A: ADD→STORE(1)
B:    ADD→STORE(1)
结果: 1 (错误!)
```
