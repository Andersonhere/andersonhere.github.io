> **核心主题**：引入持久性的概念

## 本章要点

### 什么是持久性？

持久性是指数据在系统重启后仍然保持可用的能力。

### 为什么需要持久性？

- **易失性存储**：内存断电后数据丢失
- **非易失性存储**：磁盘/SSD 可长期保存数据

### 持久性的挑战

| 挑战 | 说明 |
|------|------|
| 性能 | 磁盘 I/O 比内存慢很多 |
| 可靠性 | 硬件故障会导致数据丢失 |
| 一致性 | 崩溃可能使数据不一致 |

### 存储层次

```
┌──────────────────────┐
│   CPU 寄存器         │ 最快，最小
├──────────────────────┤
│   CPU 缓存 (L1/L2/L3)│
├──────────────────────┤
│   内存               │
├──────────────────────┤
│   固态硬盘 (SSD)     │
├──────────────────────┤
│   机械硬盘 (HDD)     │
├──────────────────────┤
│   网络存储           │ 最慢，最大
└──────────────────────┘
```
