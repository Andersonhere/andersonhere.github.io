> **核心主题**：一个真实的虚拟内存系统案例

## 本章大纲

### 23.1 背景

**VAX/VMS**：DEC 公司 1970 年代的操作系统

**特点**：
- 32 位地址空间
- 页大小 512 字节
- 早期的虚拟内存实现

### 23.4 页替换

**VMS 页替换特点**：

| 特点 | 说明 |
|------|------|
| FIFO 替换 | 简单高效 |
| 二次机会 | 避免替换活跃页 |
| 后台换出 | 减少延迟 |

**二次机会算法**：

```
检查页面:
  访问位 = 1 → 清零，给第二次机会
  访问位 = 0 → 替换
```

### 23.5 其他漂亮的虚拟内存技巧

#### 按需置零

```
分配内存时不立即清零
第一次访问时触发异常
异常处理中清零
```

#### 写时复制 (COW)

```
fork() 时:
  1. 父子进程共享物理页
  2. 页标记为只读
  3. 写入时触发异常
  4. 复制页，各自独立
```

#### 预读

```
读取一页时，预读后续页
利用磁盘顺序读取优势
```

### 23.6 小结

**VMS 的创新**：

| 技术 | 影响 |
|------|------|
| 二次机会 FIFO | 简化实现，接近 LRU |
| 按需置零 | 减少不必要操作 |
| 写时复制 | 高效的 fork |
| 预读 | 减少页错误次数 |
