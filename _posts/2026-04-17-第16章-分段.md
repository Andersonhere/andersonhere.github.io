> **核心问题**：如何减少内存碎片并支持稀疏地址空间？

## 本章大纲

### 16.1 分段：泛化的基址/界限

**核心思想**：每个逻辑段（代码、栈、堆）独立管理

```
地址空间
┌─────────┐
│   栈    │ → 段寄存器: 基址3, 界限3
├─────────┤
│   堆    │ → 段寄存器: 基址2, 界限2
├─────────┤
│   数据  │ → 段寄存器: 基址1, 界限1
├─────────┤
│   代码  │ → 段寄存器: 基址0, 界限0
└─────────┘
```

**段选择**：虚拟地址高位确定段

```
虚拟地址: [ 段号 | 偏移量 ]

段号 = 虚拟地址 >> 段偏移位数
偏移 = 虚拟地址 & 偏移掩码
```

### 16.2 我们引用哪个段

**显式 vs 隐式**：

| 方式 | 说明 |
|------|------|
| 显式 | 虚拟地址高位指定段 |
| 隐式 | 根据指令类型判断（代码/数据/栈） |

### 16.3 栈怎么办

**问题**：栈向低地址增长

**解决方案**：增加增长方向标志位

```
栈段配置:
  基址 = 栈区最高地址
  界限 = 栈区大小
  增长方向 = 向下

有效地址检查:
  基址 - 偏移 >= 基址 - 界限
```

### 16.4 支持共享

**代码共享**：多个进程共享同一段

```
进程 A 代码段: 基址=0x1000, 只读
进程 B 代码段: 基址=0x1000, 只读  (指向同一物理内存)
```

**段权限位**：

| 位 | 含义 |
|----|------|
| R | 可读 |
| W | 可写 |
| X | 可执行 |

### 16.5 细粒度与粗粒度的分段

| 类型 | 段数 | 特点 |
|------|------|------|
| 粗粒度 | 少（代码/数据/栈） | 管理简单 |
| 细粒度 | 多（每个对象） | 灵活但开销大 |

### 16.6 操作系统支持

**操作系统职责**：

1. **段表管理**：维护段寄存器表
2. **内存分配**：为每个段分配物理空间
3. **上下文切换**：保存/恢复段寄存器
4. **碎片整理**：压缩内存（困难）

### 16.7 小结

**分段优缺点**：

| 优点 | 缺点 |
|------|------|
| 支持稀疏地址空间 | 外部碎片 |
| 易于共享 | 段大小可变，管理复杂 |
| 代码保护 | 内存压缩困难 |
