## 本章大纲

```
        ┌─────────────────────────────────┐
```

### 36.2 标准设备

**设备组成**：

| 组件 | 功能 |
|------|------|
| 控制器 | 控制设备操作 |
| 数据寄存器 | 存储数据 |
| 状态寄存器 | 指示设备状态 |
| 命令寄存器 | 接收命令 |

### 36.3 标准协议

**轮询方式**：

```c
while (STATUS == BUSY)
    ; // 等待设备就绪
write_data(DATA_REGISTER, data);
write_command(COMMAND_REGISTER, WRITE);
while (STATUS == BUSY)
    ; // 等待完成
```

### 36.4 利用中断减少 CPU 开销

**中断方式**：

```c
// 发起 I/O 后 CPU 可做其他事
async_write(device, buffer);

// I/O 完成时设备触发中断
void interrupt_handler() {
    // 处理完成事件
}
```

### 36.5 利用 DMA 进行更高效的数据传送

**DMA (Direct Memory Access)**：

```
无 DMA:
  设备 ←── CPU ──→ 内存
  (CPU 负责搬运数据)

有 DMA:
  设备 ←── DMA ──→ 内存
  (CPU 只需设置 DMA，然后做其他事)
```

### 36.6 设备交互的方法

| 方法 | 说明 |
|------|------|
| I/O 指令 | 特权指令访问设备 |
| 内存映射 I/O | 设备寄存器映射到内存地址 |

### 36.7 纳入操作系统：设备驱动程序

**驱动程序结构**：

```c
struct device_driver {
    int (*init)(void);
    int (*read)(char *buf, int size);
    int (*write)(char *buf, int size);
    int (*ioctl)(int cmd, void *arg);
};
```

### 36.8 案例研究：简单的 IDE 磁盘驱动程序

```c
// 读取磁盘扇区
void ide_read(int sector, void *buf) {
    // 1. 等待设备就绪
    while (inb(STATUS) & BUSY)
        ;
    
    // 2. 设置参数
    outb(SECTOR, sector);
    outb(COUNT, 1);
    outb(COMMAND, READ);
    
    // 3. 等待完成
    while (inb(STATUS) & BUSY)
        ;
    
    // 4. 读取数据
    insl(DATA, buf, SECTOR_SIZE/4);
}
```

### 36.9 历史记录

| 时代 | 技术 |
|------|------|
| 早期 | 轮询 |
| 中期 | 中断 |
| 现代 | DMA + 中断 |

### 36.10 小结

| 技术 | 优点 | 缺点 |
|------|------|------|
| 轮询 | 简单 | 浪费 CPU |
| 中断 | 高效 | 处理复杂 |
| DMA | 高效传输 | 需要硬件支持 |
